# Public Claims And Limitations

## Allowed Public Claims

- The project demonstrates a local governed action agent loop.
- The agent separates proposal from authorization.
- The agent records decisions in a local receipt ledger.
- The ledger can be replay-verified locally.
- The ledger can use HMAC authentication when configured.
- The agent blocks or denies secret access attempts in the demo.
- The agent detects self-authorization attempts as constitutional violations.
- The agent supports scoped review tokens.
- The agent supports local reviewer identity hardening.
- The agent can generate sandboxed code-change proposals without modifying the real repo.
- The agent can generate a Markdown/JSON release-readiness report.
- The agent can generate a local static receipt viewer.

## Forbidden / Not-Yet Claims

- Do not claim production safety.
- Do not claim complete security.
- Do not claim tamper-proof ledger.
- Do not claim legal/compliance certification.
- Do not claim external audit completion.
- Do not claim SSO/OIDC identity verification.
- Do not claim real GitHub write integration.
- Do not claim real deployment control.
- Do not claim all possible violations are detected.
- Do not claim autonomous code modification of the real repo.
- Do not claim cloud-hosted dashboard.
